# Release History

이 문서는 프로젝트 진행 과정에서 발생한 기획 변경, 기능 수정, 스펙 조정 등 초기 의도와 최종 결과물이 달라진 구체적인 배경과 변경 사유를 기록하고 버전별 릴리즈 상태를 관리하는 문서입니다.

---

## ⚙️ 버저닝 규칙 (Versioning Policy)
- **첫 번째 자리 (Major)**: 프로젝트 전체 아키텍처나 핵심 구성이 크게 바뀔 정도의 메이저 업데이트 (예: 전체 다국어화 프레임워크 연동 등)
- **두 번째 자리 (Minor)**: 새로운 핵심 기능의 추가 또는 기존 기능이 완전히 수정 및 변경될 정도의 업데이트 (예: 역할 CRUD 및 드롭다운 대체 등)
- **세 번째 자리 (Patch)**: UI 개선, 사용성 보완, 단순 버그 수정 등 사소한 유지보수 업데이트 (예: Portal 오버레이 개선, 대소문자 미스매치 디버깅 등)

## 🚀 릴리즈 상세 이력

### v1.1.1 (2026-06-08) - 오버레이 Portal 전환 & 상/하(Up/Down) 동적 제어 탑재 및 프로젝트 status 디버깅
모달이나 스크롤 패널 안에서 오버레이(드롭다운, 달력, 팝오버) 하단이 잘리는 문제를 React Portal로 전격 해결하고, 화면 하단 영역 클릭 시 오버레이가 위 방향(Upward)으로 열리게 고도화하였으며, 프로젝트 수정 시 status 변경 디자인 깨짐 현상을 근본적으로 디버깅한 마일스톤입니다.

*   **공통 오버레이 컴포넌트(Select, BasePopover, CalendarPicker) React Portal 적용 및 동적 방향 제어**
    *   **초기 의도**: 드롭다운 및 날짜 선택 캘린더, 필터 팝오버 등이 각각의 부모 relative 컨테이너 내부에 렌더링되게 구현되어 모달창 하단이나 스크롤바에 가로막힘.
    *   **변경 결과**: React Portal(`createPortal`)을 활용해 오버레이 요소들을 모두 `<body>` 직속으로 렌더링하도록 전환하고, fixed 좌표를 통해 밀착 배치함. 추가적으로, 화면 세로 정중앙을 기준으로 아래쪽에 위치하면 오버레이가 위 방향(Upward, `translateY(-100%)`)으로 열리고 마진이 `-8px`로 지정되도록 동적 방향 제어 로직 탑재.
    *   **변경 사유**: 모달 창 내부 또는 좁은 화면 영역에서 사용 시 하단 정보가 가려지거나 잘리던 UI/UX적 한계를 극복하고 사용성을 극대화하기 위함.
*   **프로젝트 status 변경 시 디자인 깨짐 및 레이아웃 흔들림 디버깅**
    *   **초기 의도**: 수정 화면에서 status를 변경해 백엔드 API로 수정 요청 송신.
    *   **변경 결과**: 수정 상태값의 대소문자 미스매치를 방지하고자 실제 DB 규격(`Active`, `At Risk`, `Completed`, `On Hold` 등 PascalCase)과 매핑을 맞추었으며, 세그먼트 버튼 비선택 시에도 투명 테두리(`border-transparent`)를 할당하여 레이아웃 흔들림(Layout Shift)을 잡음. 넓이가 좁아질 때 버튼이 찢어지던 문제를 방지하고자 `flex-nowrap`을 적용함.
    *   **변경 사유**: 수정 후 대시보드로 복귀 시 상태 데이터 불일치로 뱃지 디자인 및 다국어 텍스트가 깨지던 오류를 해결하고, 라디오 클릭 시 테두리 크기 변화로 발생하던 미세 레이아웃 떨림 현상을 차단하기 위함.

### v1.1.0 (2026-06-08) - 프로젝트 역할 드롭다운 선택화 & 설정 페이지 CRUD 도입
텍스트 수동 입력을 통한 역할 지정 방식을 사전 정의된 표준 역할 목록 선택 방식으로 대체하고, 이를 유기적으로 관리할 수 있도록 변경하였습니다.

*   **프로젝트 역할(Project Role) 선택 및 관리 기능 고도화**
    *   **초기 의도**: `Manage Team` 모달 및 `New Project` 모달 리소스 할당 시, 각 인원에 대한 역할을 텍스트 인풋(Key-in) 방식으로 직접 입력하도록 설계.
    *   **변경 결과**: `Settings > Workspace Setup` 화면에 `Project Roles` 관리 카드를 신설하여 역할을 추가 및 삭제(CRUD)할 수 있도록 구현하고, 할당 모달 내 역할 입력란을 해당 역할 목록이 바인딩된 `<Select>` 드롭다운 컴포넌트로 전면 대체.
    *   **변경 사유**: 텍스트 수동 입력 시 발생하는 데이터의 오타 및 비일관성(예: 'Dev', 'Developer', 'Frontend' 등의 파편화) 문제를 방지하고, 프로젝트 조직 내 역할 체계를 사전에 통제하여 데이터 표준화를 달성하기 위함.

### v1.0.0 (2026-06-08) - 프로젝트 다국어화 & 서비스 안정화
글로벌 협업 환경 지원을 위한 다국어(i18n) 연동과 시스템 안정성 확보에 집중한 마일스톤입니다.

*   **프로젝트 전체 다국어화 (KO / EN) 연동**
    *   **초기 의도**: 하드코딩된 한국어 및 영어가 혼재된 UI 제공.
    *   **변경 결과**: `react-i18next`를 적용하여 대시보드, 프로젝트, 자산, 임직원 관리, 캘린더, 설정 등 모든 메뉴의 텍스트와 안내 경고 메시지를 다국어 사전화(`ko.json`, `en.json`)하고 헤더를 통해 실시간 언어 전환이 가능하도록 구현.
    *   **변경 사유**: 글로벌 확장성 확보 및 사용자층 확대를 위해 영문/국문 번역의 일관된 제공이 필수적이라 판단하여 전격 리팩토링 진행.
*   **Vite 빌드 성능 및 타입 안정성 강화**
    *   TypeScript 컴파일 에러를 모두 수정하고 프로덕션 빌드 성공 검증 완료.

### v0.3.0 (2026-05-30) - 장비 관리(IT Assets) 고도화
물리적 하드웨어의 입출고 상태와 장비 운용 현황을 정교하게 관리하도록 화면 및 데이터 싱크 로직을 고도화했습니다.

*   **장비 자동 상태 표시 로직 도입 (기획 대폭 변경)**
    *   **초기 의도**: 장비 등록 및 관리 시, '배정됨', '배정 가능', '수리중', '수리 필요' 등의 상태 값을 사용자가 직접 선택하도록 설계.
    *   **변경 결과**: '하드웨어 상태(Hardware Status)'를 사용자가 수동 선택하지 못하는 **읽기 전용 표시 영역**으로 변경. 대신 배정된 사용자(User) 유무와 '기기 가동 상태(Normal / Maintenance / Needs Repair)'를 조합하여 장비 상태를 자동 판별해 표시하도록 수정.
        *   배정 사용자 있음 + 기기 정상 ➡️ **Assigned (배정됨)**
        *   배정 사용자 없음 + 기기 정상 ➡️ **Available (배정 가능)**
        *   기기 고장/정비 중 ➡️ **Needs Repair / Maintenance** 자동 노출
    *   **변경 사유**: 사용자가 임의로 배정 상태를 수정할 경우 실제 인사 정보와 데이터 불일치가 발생할 수 있으므로, 시스템 신뢰성을 보장하기 위해 데이터 종속성(직원 배정 여부 및 실제 기기 가동 여부)에 따라 상태가 자동 결정되도록 안전 장치 도입.
*   **장비 인벤토리 필터 뷰(Filter View) 및 팝오버 구현**
    *   장비 유형(Type) 및 상태(Status)별 조회가 가능한 복합 필터 팝오버 구현.
    *   **UI/UX 개선**: 팝오버 열기 버튼 하단 우측에 팝오버 창이 안정적으로 정렬되도록 레이아웃 및 `z-index` 흐름 최적화.

### v0.2.0 (2026-05-15) - 불필요 기능 제거 및 테마 단일화
비즈니스 요구사항을 간소화하고, 복잡도를 낮추어 코어 기능에 집중하도록 변경된 마일스톤입니다.

*   **다중 테마(Theme) 연동 제거 및 단일 테마(다크 모드 전용) 최적화**
    *   **초기 의도**: 라이트/다크 모드 전환 버튼을 배치하여 다양한 디자인 테마 제공 시도.
    *   **변경 결과**: 헤더 내 테마 변경 버튼을 전면 삭제하고, 다중 테마 지원을 위해 작성되었던 불필요한 CSS 변수 및 CSS 구조를 단일 다크 모드에 맞춰 대폭 단순화 및 최적화.
    *   **변경 사유**: 프리미엄 다크 테마(Dark-only) 전용 디자인 시스템에 집중하여 디자인 아이덴티티를 명확히 하고, CSS 파편화로 인한 스타일 유지보수 비용을 줄이기 위해 단일화 결정.
*   **Welfare(임직원 복지) 기능 및 Export Inventory 삭제**
    *   **초기 의도**: 장비 메뉴 쪽에 직원 복지 관련 혜택 관리 기능을 추가하려 함.
    *   **변경 결과**: Employee Welfare 탭 및 관련 컨트롤러를 완전히 제거. 인벤토리 데이터 내보내기(Export Inventory) 버튼 삭제.
    *   **변경 사유**: 본 서비스의 도메인은 '프로젝트 관리 및 IT 자원 수명 주기 관리'이므로, 장비 관리 영역에 복지 성격의 기능이 혼재되는 것은 도메인 집중도를 흐린다고 판단하여 삭제. 비품의 무단 반출 등을 방지하기 위해 불필요한 인벤토리 엑셀 유출 통로(Export)를 차단하여 보안성 강화.
